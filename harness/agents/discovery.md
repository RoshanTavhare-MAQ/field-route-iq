# Discovery Agent — Instructions

## Role
Scan the repository and produce a structured, evidence-based list of incomplete
features. Write results to `harness/state/feature-gaps.json`.

---

## Inputs to read

1. `SPEC.md` — authoritative feature specification (primary source)
2. `README.md` — high-level status and missing module list
3. `RULES.md` — scoring rules (what counts as done)
4. `src/data/index.ts` — all available data loaders and types
5. `src/pages/OrderPage.tsx` — known inline TODO
6. `src/legacy/pricingV1.ts` — reference only; do NOT treat its logic as correct
7. `docs/NOTES.md` — meeting notes; treat as unreliable, SPEC overrides

## Inputs to ignore

- `src/legacy/discountMatrix.ts` — not part of the SPEC.md pricing model
- Any comment in `pricingV1.ts` about rules — its semantics conflict with SPEC.md

---

## Signals to look for

| Signal type | Where to look |
|---|---|
| Missing files | `src/pricing/`, `src/audit/`, `src/settlement/` — do these directories exist? |
| TODO/FIXME comments | `src/pages/OrderPage.tsx` line 32, any other source files |
| Skipped / failing tests | `npm run test` output |
| Placeholder implementations | Functions that throw or return empty results |
| Spec vs implementation gaps | Compare SPEC.md sections to what exists in `src/` |
| Unused modules | `src/legacy/` files that nothing imports |

---

## Output format

Update `harness/state/feature-gaps.json` with an array of gap objects.  
Each gap must include these fields:

```json
{
  "id": "short-kebab-id",
  "name": "Human-readable feature name",
  "status": "missing | incomplete | partial | unverified",
  "scored": true,
  "priority": 1,
  "evidence": ["list of evidence strings with file:line refs"],
  "affectedFiles": ["src/pricing/engine.ts"],
  "expectedBehavior": "What SPEC.md says this should do",
  "currentGap": "What is missing right now",
  "suggestedApproach": "High-level implementation approach",
  "verificationStrategy": "How to validate when done",
  "specSections": ["§2", "§3", "§4", "§5", "§6", "§7"],
  "dependencies": ["other-gap-id"],
  "completionPlan": null,
  "reviewerApproved": false,
  "blockingQuestions": []
}
```

---

## Confidence rules

- Mark `scored: true` only for features explicitly listed in `RULES.md` as deliverables.
- Mark `status: "missing"` when the file does not exist.
- Mark `status: "incomplete"` when a file exists but has a TODO/placeholder.
- Do not flag style preferences or optional enhancements as gaps.
- If a gap has conflicting signals between `docs/NOTES.md` and `SPEC.md`, note it in `blockingQuestions` and defer to `SPEC.md`.

---

## False-positive avoidance

- `src/legacy/` files intentionally exist as reference; they are not gaps.
- `OrderPage.tsx` promotion wiring is documented as "not scored" in SPEC.md §8.
- The `COST.txt` file will be auto-generated by `agent-run.mjs`; its absence is not a gap.
- Do not flag missing test files — RULES.md says the agent must not write tests.
